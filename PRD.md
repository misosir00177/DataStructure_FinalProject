# Product Requirements Document (PRD)
## 1. 專案概述 (Project Overview)
本專案為一款「二手物品以物易物」的 Web App 前端原型。主打無金錢交易、類似交友軟體的快節奏滑動卡片配對體驗。本原型專為課堂展示設計，全程在前端運行，無真實後端與資料庫，並透過展示特定資料結構（Queue/Array, Hash Map/Set）的運作來滿足技術評分需求。

## 2. 目標受眾與情境 (Target Audience & Use Cases)
- 衣物過剩的大學生：需要出清只穿過幾次的衣服，並希望能換到其他二手衣物。

- 搬家換宿的年輕人：需要處理不再使用的雜物，同時以低預算（無金錢）獲取二手家具。

- 盲盒與周邊收藏家：櫃子空間不足，希望能將完好的公仔交給同好，並在卡片池中加入「未知盲盒」的特殊卡片以滿足刺激感。

## 3. 核心功能與介面需求 (Core Features & UI)
- 登入與主頁 (Fake Login & Dashboard)： 提供一鍵進入的假登入畫面，以及顯示使用者目前「可交換物品」的個人首頁。

- 模擬上傳與 AI 分析 (Mock Upload & AI Tagging)： 點擊上傳按鈕後，觸發 setTimeout 模擬進度條，隨機或固定生成 3-5 個物品標籤（例如：#3C, #九成新, #原宿風）。

- 探索與篩選器 (Explore & Filter)： 介面上方提供標籤 Toggle 按鈕。點擊後能即時過濾下方的卡片堆。

- 滑動配對介面 (Swipe Deck)： 畫面中央顯示物品卡片。實作向左滑動（拒絕）與向右滑動（想交換）的動畫效果。

- 配對成功畫面 (Match & Chat)： 當滑到特定預設卡片並右滑時，強制觸發「It's a Match!」全螢幕特效，並進入純文字的假約定地點聊天室。

## 4. 資料結構應用說明 (Data Structure Implementation)
- Queue (佇列) / Array： 用於管理「待滑動的物品卡片」。當使用者設定好篩選器後，符合條件的物品資料物件會被推入 - Queue 中。使用者每滑動一次，就從 Queue 首端 shift() 取出一張卡片渲染在畫面上，完美解釋先進先出（FIFO）或卡片堆疊的概念。

- Hash Set (集合)： 用於實作「標籤篩選器」。將使用者選擇的過濾標籤存入 Set，每次載入卡片時，利用 Set 的高效率查找特性，比對卡片陣列中的標籤與 Set 中的標籤是否交集，以決定該卡片是否進入 Queue。

## 5. 黃金展示路線 (Demo Golden Path)
Step 1： 開啟 index.html，點擊「快速登入」。

Step 2： 進入上傳頁面，點擊「模擬拍攝」，畫面顯示掃描動畫，2秒後跳出成功提示並自動標註 #木製家具, #九成新。

Step 3： 系統跳轉至滑動首頁。此時跳出提示框要求使用者「選擇您要尋找的物品類型」。

Step 4： 使用者點選 #衣物 與 #盲盒（此時展示 Hash Set 過濾邏輯）。

Step 5： 畫面上出現第一張卡片（衣服），使用者向左滑（忽略）。

Step 6： 出現第二張卡片（盲盒），使用者向右滑（喜歡）。

Step 7： 系統觸發「It's a Match!」畫面，顯示使用者的 #木製家具 與對方的 #盲盒 配對成功。

Step 8： 自動跳轉至假聊天室，畫面上已預載一條訊息：「你好！我對你的木製家具很有興趣，約捷運站面交嗎？」
